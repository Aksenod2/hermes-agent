# Транскрибатор Аудио

## Задача
Получи аудиофайл → транскрибируй через Polza.ai API → сохрани текст + краткое содержание.

## Требования
- **API ключ:** Переменная окружения `POLZA_API_KEY` должна содержать ключ от Polza.ai
- **Поддерживаемые форматы:** mp3, wav, ogg, m4a, flac, opus

## API Спецификация

### Endpoint
```
POST https://polza.ai/api/v1/audio/transcriptions
```

### Headers
```
Authorization: Bearer {POLZA_API_KEY}
Content-Type: application/json
```

### Request Body
```json
{
  "file": "data:audio/<тип>;base64,<base64_encoded_audio>",
  "model": "openai/whisper-1",
  "language": "ru",
  "response_format": "verbose_json"
}
```

### Response
```json
{
  "text": "Полный текст транскрибации...",
  "duration": 123.45,
  "language": "ru"
}
```

## Процесс транскрибации

### Шаг 1: Подготовка аудио файла
1. Получи путь к аудио файлу от пользователя
2. Прочитай файл в бинарном режиме
3. Определи MIME тип файла (audio/mpeg, audio/wav, и т.д.)
4. Закодируй содержимое файла в base64
5. Сформируй data URI: `data:{mime_type};base64,{base64_data}`

### Шаг 2: Отправка запроса
1. Проверь наличие `POLZA_API_KEY` в переменных окружения
2. Сформируй POST запрос с headers и body согласно спецификации выше
3. Отправь запрос с timeout 120 секунд (большие файлы могут обрабатываться долго)

### Шаг 3: Обработка результата
1. Получи JSON ответ от API
2. Извлеки текст транскрибации из поля `"text"`
3. Создай краткое содержание (summary):
   - Если текст короткий (<300 символов) - summary не нужно
   - Если длинный - укажи длину, основные темы, кто говорит (если понятно)

### Шаг 4: Сохранение результата
1. Создай файл с именем `{исходное_имя_аудио}.txt` в той же папке
2. Сохрани в файл:
   ```
   Транскрибация:
   {полный_текст}

   ---

   Краткое содержание:
   {summary}
   ```
3. Верни пользователю:
   - Полный текст транскрибации
   - Путь к сохраненному файлу
   - Краткое содержание

## Пример использования

**Пользователь:** Транскрибируй этот аудио файл: `/path/to/meeting.mp3`

**Агент выполняет:**
1. Читает файл `/path/to/meeting.mp3`
2. Конвертирует в base64
3. Отправляет на Polza.ai API
4. Получает текст транскрибации
5. Сохраняет в `/path/to/meeting.txt`
6. Возвращает текст + summary пользователю

## Обработка ошибок

- **Файл не найден:** Сообщи пользователю что файл не существует
- **API ключ отсутствует:** Попроси пользователя установить `POLZA_API_KEY`
- **Ошибка API:** Покажи сообщение об ошибке от API
- **Неподдерживаемый формат:** Попроси файл в поддерживаемом формате

## Параметры языка

По умолчанию используй `"language": "ru"` для русского языка.

Другие языки:
- `"en"` - английский
- `"uk"` - украинский
- `"auto"` - автоопределение

Спрашивай пользователя о языке только если не уверен.
